# CLI モード

法律アシスタントのコマンドラインインターフェース（CLI）を使用すると、ターミナルから直接法的な質問ができます。

## 基本的な使い方

### インタラクティブモード

```bash
# CLI を対話モードで起動
pnpm cli
```

起動後、プロンプトが表示されるので、自由に質問を入力できます：

```
法律アシスタントへようこそ！質問を入力してください（終了するには 'exit' または Ctrl+C）:

> 労働契約満了後の更新なしには賠償が必要ですか？
```

### 直接質問モード

```bash
# 引数を直接渡して質問
pnpm cli "労働契約紛争の対応方法は？"
```

### 開発モード

```bash
# 変更を自動的にリロード
pnpm dev
```

## 使用例

### シンプルな法的質問

```bash
pnpm cli "労働基準法における休憩時間について教えてください"
```

**出力例**:
```
労働基準法第 34 条では、使用者は労働時間が 6 時間を超える場合には少なくとも 45 分、
8 時間を超える場合には少なくとも 1 時間の休憩時間を労働時間の途中に与えなければなりません...
```

### 複雑な法的相談

```bash
pnpm cli "会社が突然解雇を通告してきました。違法解雇の可能性はありますか？"
```

### 条文検索

```bash
pnpm cli "労働契約法第 16 条の条文を検索してください"
```

## オプション

### ヘルプの表示

```bash
pnpm cli --help
```

### バージョンの確認

```bash
pnpm cli --version
```

## 高度な機能

### パイプ入力の使用

```bash
# ファイルから質問を読み込む
echo "労働契約の更新義務について" | pnpm cli

# ファイルの内容を処理
cat questions.txt | pnpm cli
```

### 出力のリダイレクト

```bash
# 結果をファイルに保存
pnpm cli "労働契約について" > answer.txt

# JSON 形式で出力
pnpm cli "労働契約について" --json > answer.json
```

## 設定

### 環境変数によるカスタマイズ

`.env` ファイルで以下の設定を調整できます：

```bash
# モデルの変更
LLM_MODEL_NAME=gpt-4
LLM_TEMPERATURE=0.7

# 応答の長さ
LLM_MAX_TOKENS=2048

# ロギングレベル
LOG_LEVEL=debug
```

### プロンプトのカスタマイズ

`src/chains/legalChain.ts` のプロンプトテンプレートを編集して、回答のスタイルを調整できます。

## ツールの統合

CLI は以下のツールを自動的に使用して、より正確な回答を提供します：

### ドキュメント検索

関連する法律条文や判例を自動的に検索します。

### ウェブ検索

最新の法令改正や判例をウェブから検索します（設定により有効/無効）。

## トラブルシューティング

### API エラー

```bash
Error: API request failed
```

**解決方法**:
1. API キーが正しいことを確認
2. ネットワーク接続を確認
3. レート制限に達していないことを確認

### モデルの読み込みエラー

```bash
Error: Model not found
```

**解決方法**:
```bash
# Ollama モデルをプル
ollama pull gpt-oss:20b

# または .env でモデル名を変更
LLM_MODEL_NAME=gpt-3.5-turbo
```

### メモリ不足エラー

```bash
Error: JavaScript heap out of memory
```

**解決方法**:
```bash
# Node.js のメモリ制限を増加
NODE_OPTIONS="--max-old-space-size=4096" pnpm cli
```

## ベストプラクティス

### 1. 具体的な質問をする

❌ 悪い例: 「労働法について」
✅ 良い例: 「労働基準法第 34 条の休憩時間規定について」

### 2. 文脈を提供する

❌ 悪い例: 「解雇されました」
✅ 良い例: 「正社員として 3 年勤務後、予告なく解雇されました」

### 3. フォローアップ質問

最初の回答に対してさらに深く掘り下げる質問をすると、より詳細な情報が得られます。
