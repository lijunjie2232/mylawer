セキュリティは法律アシスタントの最優先事項です。法的データと AI セッションを安全に保つために、複数の保護レイヤーを実装しています。

## 認証システム

法律アシスタントは、ステートレスな認証のために **JSON Web Tokens (JWT)** を使用しています。

### ユーザーロール
1.  **一般ユーザー (User)**: セッションの作成、AI とのチャット、および自身の履歴管理が可能です。
2.  **管理者 (Admin)**: すべてのユーザーの管理、システム統計の表示、およびグローバル設定の構成が可能です。

### 設定
これらは `.env` で設定します：
- **`JWT_SECRET`**: トークンの署名に使用される、強力でユニークな文字列。決して共有しないでください。
- **`JWT_EXPIRES_IN`**: セッションの有効期間 (デフォルト 24 時間)。

## 初期管理者のセットアップ

アプリケーションを最初に実行したとき、`INIT_ADMIN=true` が設定されていると、デフォルトの管理者が作成されます。

- **デフォルトメール**: `admin@example.com`
- **デフォルトパスワード**: `Admin123!@#`

**必要なアクション**: 初回ログイン後、ウェブ UI のプロファイルセクションからすぐに管理者のパスワードを変更してください。

## CORS とネットワークセキュリティ

### Cross-Origin Resource Sharing (CORS)
開発環境では、サーバーはすべてのオリジンからのリクエストを許可します。本番環境では、特定のフロントエンドドメインに制限する必要があります。

```typescript
// src/server.ts での設定例
if (config.app.env === 'production') {
  app.use(cors({ origin: 'https://your-legal-app.com' }));
}
```

### LINE 署名検証
LINE Bot 統合では、LINE からのすべてのリクエストが `LINE_CHANNEL_SECRET` に対して検証されます。これにより、権限のない第三者が AI をトリガーしたり、Webhook エンドポイント経由でチャットデータにアクセスしたりするのを防ぎます。

## データ暗号化

- **パスワード**: モダンな暗号アルゴリズムを使用してハッシュ化された文字列として保存されます。プレーンテキストのパスワードは保存しません。
- **トークン**: LLM プロバイダーの機密性の高い API キーは、環境変数からメモリに保存され、ログに記録されたり UI に公開されたりすることはありません。

## ベストプラクティス

1.  **HTTPS**: 本番環境では、必ず SSL/TLS プロキシ (Nginx や Caddy など) の背後で法律アシスタントを実行してください。
2.  **環境の分離**: 開発用と本番用で異なる `.env` ファイルを使用してください。
3.  **監査ログ**: `LOG_LEVEL=info` の出力を監視して、ログインや管理アクションを追跡してください。
