法律アシスタントでは、AI エージェントが利用できるツールを正確に制御できます。これはツールのホワイトリスト化を通じて管理されます。

## ホワイトリストの原則

デフォルトでは、法律アシスタントは接続された MCP サーバーによって提供されるすべてのツールの読み込みを試みます。しかし、セキュリティ、パフォーマンス、または UI の明確さのために、これらを制限したい場合があります。

### `ALLOWED_MCP_TOOLS`

この環境変数を使用して、許可されるツールを正確に指定します。フォーマットは `serverName_toolName` です。

```bash
# 例: 法律検索と基本的なウェブ検索のみを許可
ALLOWED_MCP_TOOLS=legal-mcp_search_laws,webmcp_search
```

## 利用可能なツールリファレンス

### legal-mcp (日本の法律)

| ツール識別子 | ユースケース |
| :--- | :--- |
| `legal-mcp_search_laws` | 法的データベースに対するセマンティック検索。 |
| `legal-mcp_get_law_by_id` | 特定の条文の全文取得。 |
| `legal-mcp_get_cluster_status`| 法的検索インデックスの健全性チェック。 |

### webmcp (インターネットコンテキスト)

| ツール識別子 | ユースケース |
| :--- | :--- |
| `webmcp_search` | 検索エンジンによる標準的なキーワード検索。 |
| `webmcp_parse_url` | 特定のウェブサイトからのテキスト抽出。 |
| `webmcp_deep_search` | 多段階の調査とレポート合成。 |

## なぜツールをフィルタリングするのか？

1.  **コスト制御**: 一部のツール (`deep_search` など) は、エージェントが複数のページを分析するため、より多くのトークンを消費する可能性があります。
2.  **正確性**: エージェントを専門的なツール (`legal-mcp` など) に制限することで、一般的なブログではなく、高品質な法的ソースに集中させることができます。
3.  **セキュリティ**: 機密性の高いメタデータを公開したり、不正なアクションを許可したりする可能性のあるツールへのアクセスをエージェントから防ぎます。

## 高度な設定: カスタムツールの追加

新しいツールを追加するには：
1.  新しい MCP サーバーを `third/` ディレクトリに配置します。
2.  `src/mcp/mcpInitializer.ts` を更新して、新しいサーバーを含めます。
3.  新しいツールを `ALLOWED_MCP_TOOLS` ホワイトリストに追加します。

## トラブルシューティング

- **ツールが表示されない**: `ALLOWED_MCP_TOOLS` のスペルを確認してください。正しいサーバープレフィックス (`webmcp_` や `legal-mcp_`) を使用していることを確認してください。
- **初期化エラー**: サーバーログを確認してください。MCP サーバーの接続に失敗した場合、ホワイトリストの設定に関わらず、そのサーバーのツールは利用できません。
