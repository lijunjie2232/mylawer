法律アシスタントは、AI エージェントに外部機能を提供するために **Model Context Protocol (MCP)** を使用しています。ツールをメインアプリケーションから切り離すことで、システムの軽量性と拡張性を維持しています。

## 現在の MCP サーバー

デフォルトで 2 つの主要な MCP サーバーが同梱されています：

### 1. legal-mcp (日本の法律検索)
このサーバーは法的インテリジェンスの中核です。数千の日本の法律や規制を含む専用の検索エンジン (Elasticsearch) に接続します。

- **`legal-mcp_search_laws`**: 法的データベースに対してセマンティック検索またはキーワード検索を実行します。
- **`legal-mcp_get_law_by_id`**: 特定の法律の完全でフォーマットされたテキストを取得します。
- **`legal-mcp_get_index_state`**: (管理者用) 法的データベースの統計情報を提供します。

### 2. webmcp (一般的なウェブインテリジェンス)
法的データベースに回答がない場合、またはユーザーが最近の出来事やニュースについて尋ねた場合、アシスタントは `webmcp` を使用します。

- **`webmcp_search`**: 複数のエンジン (Google、Tavily など) を使用してインターネットを検索します。
- **`webmcp_parse_url`**: 特定のウェブページのコンテンツを取得してクリーンアップします。
- **`webmcp_deep_search`**: 検索を実行し、複数の結果を分析し、調査レポートを合成する高度なツール。

## ツールの読み込み方法

起動時に `mcpInitializer.ts` サービスが以下の処理を行います：
1.  `third/` ディレクトリ内の設定済み MCP サーバーを特定します。
2.  標準入出力 (Node.js 子プロセス) 経由でそれらに接続します。
3.  マニフェストを検査して、提供されているツールを確認します。
4.  環境変数 `ALLOWED_MCP_TOOLS` に基づいて、これらのツールを**フィルタリング**します。
5.  それらを LangChain 互換のツールに適合させます。

## なぜ MCP なのか？

- **セキュリティ**: MCP サーバーは独自のプロセスで実行されます。権限 (例: ファイルシステムの読み取り専用) を制限できます。
- **相互運用性**: コミュニティによって構築された任意の MCP サーバー (GitHub、Slack、データベースコネクタなど) を使用できます。
- **スケーラビリティ**: 重い処理 (複雑な法律の解析など) は、メインアプリケーションのエベントループではなく、MCP サーバープロセスで行われます。

## ツール管理

`.env` 内の `ALLOWED_MCP_TOOLS` 変数を使用して、どのアクティブなツールを使用するかを制御できます：

```bash
# 法律検索とウェブ解析のみを許可
ALLOWED_MCP_TOOLS=legal-mcp_search_laws,webmcp_parse_url
```

詳細なフィルタリングについては、[ツール管理ガイド](/ja/config/tools/) を参照してください。
