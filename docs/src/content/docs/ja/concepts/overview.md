法律アシスタントは、インテリジェントな法的相談を提供するために設計された、モダンでモジュール化されたアプリケーションです。最新の AI オーケストレーションと分散型ツールプロトコルを活用しています。

## ハイレベルアーキテクチャ

システムは主に 4 つのレイヤーで構成されています：

1.  **インターフェースレイヤー**: React ウェブ UI、CLI、および LINE Bot 連携。
2.  **サーバーレイヤー**: 認証、セッション管理、およびルーティングを処理する Node.js/Express サーバー。
3.  **エージェントレイヤー**: **LangChain** と **LangGraph** を基盤とし、会話の流れを管理してツールを使用するかどうかを判断します。
4.  **ツールレイヤー (MCP)**: 法的ドキュメント検索やウェブスクレイピングなどの専門的な機能を提供する、独立した **Model Context Protocol** サーバー。

## 主要テクノロジー

### LangChain & LangGraph
アシスタントの「脳」は LangChain エージェントです。複雑でステートフルなマルチターンの会話とツール呼び出しループを管理するために LangGraph を使用しています。これにより、エージェントは繰り返し情報を検索し、解析し、回答を洗練させることができます。

### Model Context Protocol (MCP)
ツールをハードコーディングする代わりに、法律アシスタントは MCP を使用しています。これは、アシスタントがツールを提供する外部の「サーバー」に接続できるようにするオープンな標準です。これにより、システムは高度に拡張可能になります。新しい MCP サーバーを指すだけで、新しい機能を追加できます。

### Prisma & PostgreSQL
PostgreSQL との対話には ORM として Prisma を使用しています。これにより、以下が永続的に保存されます：
- ユーザーアカウントと認証。
- 詳細なチャット履歴。
- 管理設定。

### React (Vite)
モダンなウェブフロントエンドは React と Vite で構築されており、ストリーミングレスポンスと履歴コンテキストをサポートする、高速でレスポンスの良いチャットインターフェースを提供します。

## データフロー

1.  **ユーザー入力**: ユーザーがウェブ UI、CLI、または LINE 経由で質問を送信します。
2.  **認証**: サーバーレイヤーが JWT または LINE 署名を検証します。
3.  **エージェントの起動**: エージェントレイヤーが質問を受け取り、データベースからセッション履歴を読み込みます。
4.  **ツールの実行**: エージェントが法的文脈が必要であると判断し、`legal-mcp` サーバーのツールを呼び出します。
5.  **LLM 処理**: ツールからの結果が LLM (GPT-4、Claude など) に送られ、回答が生成されます。
6.  **ストリーミングレスポンス**: 回答がリアルタイムでユーザーにストリーミングされます。
